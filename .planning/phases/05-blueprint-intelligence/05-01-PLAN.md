  <files>src/tools/node-catalog-tools.ts, src/warudo/types.ts</files>
  <behavior>
    - extractNodeTypesFromScene extracts unique node type IDs from scene response graphs
    - extractNodeTypesFromScene handles PascalCase (Nodes, TypeId) and camelCase (nodes, typeId) fields
    - extractNodeTypesFromScene returns empty array when no graphs exist
    - extractNodeTypesFromScene counts usage per type and collects example node names
    - extractNodeTypesFromScene deduplicates by typeId
    - formatNodeTypeList formats output with type IDs, usage counts, and example names
    - formatNodeTypeList shows "No node types found" for empty array
  </behavior>
  <action>
    1. Add `NodeTypeInfo` interface to `src/warudo/types.ts`:
       ```typescript
       export interface NodeTypeInfo {
         typeId: string;
         usageCount: number;
         exampleNames: string[];
         dataInputKeys: string[];
       }
       ```

    2. Create `src/tools/node-catalog-tools.ts` with:
       - Import `extractGraphsFromScene` from `./blueprint-tools.js` (reuse existing helper)
       - Import `warudoError` from `../errors.js`
       - Import `NodeTypeInfo` from `../warudo/types.js`

       - `extractNodeTypesFromScene(response: Record<string, unknown>): NodeTypeInfo[]`
         - Call `extractGraphsFromScene(response)` to get graphs array
         - If no graphs, return empty array
         - Iterate all graphs, extract Nodes/nodes arrays
         - For each node, read TypeId/typeId, Name/name, DataInputs/dataInputs
         - Build a Map<string, { count, names, dataInputKeys }> keyed by typeId
         - Collect data input keys from DataInputs objects (Object.keys)
         - Return sorted array of NodeTypeInfo (by usageCount descending)

       - `formatNodeTypeList(types: NodeTypeInfo[]): string`
         - If empty: "No node types found in the current scene's blueprints.\n\nThis could mean no blueprints exist yet, or the scene has no graphs."
         - Otherwise: header "Node Types ({count} types found across scene blueprints):" followed by each type:
           ```
             - {typeId} (used {count}x)
               Names: {exampleNames.join(", ") || "unnamed"}
               Data Inputs: {dataInputKeys.join(", ") || "none observed"}
           ```

       - `listNodeTypesHandler(wsClient: WarudoWebSocketClient, wsUrl: string)`
         - Call wsClient.ensureConnected(), wsClient.send({ action: "getScene" })
         - Extract node types via extractNodeTypesFromScene
         - Format via formatNodeTypeList
         - Return MCP text content response
         - On error: return warudoError with connection guidance
         - Also try sending action "getNodeTypes" (speculative — may or may not exist in Warudo). If it returns an array, merge those types into the results. If it fails or returns nothing useful, silently fall back to scene-only extraction.

  </action>
