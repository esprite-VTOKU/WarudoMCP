7. Import warudoError from "./errors.js"

8. Create McpServer instance: `new McpServer({ name: "warudo-mcp", version: "0.1.0" })`

9. Load config via `loadConfig()`. Create client instances:
   - `const wsClient = new WarudoWebSocketClient(config.warudoWsUrl)`
   - `const restClient = new WarudoRestClient(config.warudoRestUrl)`
   - Do NOT call connect() -- lazy connection per user decision.

10. Register "ping" tool:
    - Name: "ping"
    - Description: "Check if the MCP server is running and can reach Warudo"
    - Input schema: {} (no parameters)
    - Handler: Try to call wsClient.ensureConnected(). If successful, return text with "pong" and connected status. If connection fails, return warudoError() with actionable message about Warudo not being reachable. Always include wsClient.getState() in the response so the LLM knows connection status.

11. Register "check_connection" tool:
    - Name: "check_connection"
    - Description: "Check connectivity to both Warudo WebSocket and REST APIs"
    - Input schema: {} (no parameters)
    - Handler: Try both wsClient.ensureConnected() and restClient.getAbout() independently. Report status of each. Use try-catch for each so one failure doesn't mask the other. Return a text summary like:
      "WebSocket: connected (ws://localhost:19053)\nREST API: connected (http://localhost:19052)\nWarudo version: {from about}"
      Or partial failure: "WebSocket: error - Could not connect...\nREST API: connected..."

12. Create main() async function:
    - Create StdioServerTransport
    - Call server.connect(transport)
    - Log to stderr: "warudo-mcp server running on stdio"

13. Call main().catch() -- log fatal error to stderr and exit(1).

14. After writing src/index.ts, run `npx tsc` to build. Verify build/index.js exists.

15. Update package.json "bin" field to point to build/index.js for potential npx usage:
