Create the following modules. CRITICAL: All import paths MUST use .js extensions (e.g., `import { Config } from "./config.js"`). Never use console.log -- only console.error for logging.

**src/config.ts:**
- Export `interface Config { warudoWsUrl: string; warudoRestUrl: string; }`
- Export `function loadConfig(): Config` reading from `process.env.WARUDO_WS_URL` and `process.env.WARUDO_REST_URL` with defaults `ws://localhost:19053` and `http://localhost:19052`

**src/errors.ts:**
- Export `function warudoError(message: string)` returning `{ content: [{ type: "text" as const, text: message }], isError: true }`
- This is the standard pattern for returning errors from MCP tool handlers without throwing

**src/warudo/types.ts:**
- Export `type ConnectionState = "disconnected" | "connecting" | "connected"`
- Export `interface WarudoWebSocketMessage { action: string; [key: string]: unknown; }` for outbound messages
- Export `interface WarudoWebSocketResponse { [key: string]: unknown; }` for inbound messages
- Keep types minimal -- they will grow in later phases

**src/warudo/websocket-client.ts:**
- Import WebSocket from "ws" (default import)
- Import ConnectionState from types
- Export class `WarudoWebSocketClient`:
  - Constructor takes `url: string`
  - Private `ws: WebSocket | null = null`, `state: ConnectionState = "disconnected"`, `connectPromise: Promise<void> | null = null`
  - `getState(): ConnectionState`
  - `async connect(): Promise<void>` -- if connected, return. If connecting, await existing promise (prevent duplicate connections). Otherwise create WebSocket, set 5-second timeout, resolve on "open", reject on timeout/error, set state throughout. Call `setupEventHandlers()` on success.
  - `async ensureConnected(): Promise<void>` -- call connect() if not connected
  - `private setupEventHandlers(ws: WebSocket)` -- on "close" set state=disconnected and ws=null and log to stderr, on "error" log to stderr
  - `async send(message: WarudoWebSocketMessage): Promise<WarudoWebSocketResponse>` -- ensureConnected first, then send JSON. For Phase 1, implement a simple request-response pattern: generate a unique requestId, send the message, wait for a response with timeout (10 seconds). Parse response from "message" event. If the WebSocket is not open when sending, throw with actionable error.
  - `close(): void` -- close WebSocket if open, set state=disconnected
  - Error messages must be actionable: "Could not connect to Warudo at {url}. Is Warudo running with WebSocket enabled?"

**src/warudo/rest-client.ts:**
- Export class `WarudoRestClient`:
  - Constructor takes `baseUrl: string`
  - `async getAbout(): Promise<unknown>` -- GET /api/about
  - `async getScenes(): Promise<unknown>` -- GET /api/scenes
  - `async openScene(sceneName: string): Promise<unknown>` -- PUT /api/openedScene with body `{ name: sceneName }`
  - Private `async get(path: string): Promise<unknown>` -- fetch with error handling. On non-ok response, throw with status, statusText, and path. On fetch failure (network error), throw with actionable message: "Cannot reach Warudo REST API at {baseUrl}. Is Warudo running?"
  - Private `async put(path: string, body: unknown): Promise<unknown>` -- same pattern with PUT method and Content-Type: application/json header
