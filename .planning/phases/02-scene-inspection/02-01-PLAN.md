    - list_assets returns warudoError() if WebSocket call fails

**Step 1: Add types to src/warudo/types.ts**

Add after existing type definitions:
```typescript

  id: string;
  name: string;
  type: string;
  active?: boolean;
}
```

**Step 2: Add list_assets tool to src/index.ts**

1. Add import for new types at the top if needed (WarudoAssetSummary)
2. Register tool:
   ```
   server.tool(
     "list_assets",
     "List all assets in the current Warudo scene with their names, types, and entity IDs",
     {
       type_filter: z.string().optional().describe("Filter assets by type (e.g., 'CharacterAsset', 'CameraAsset'). Omit to list all assets."),
     },
     handler
   )
   ```
3. Handler implementation:
   a. Call `wsClient.send({ action: "getScene" })` to get full scene data
   b. Extract assets from response -- try `response.assets` as an array. The exact structure depends on Warudo's response format, so be defensive:
      - If response has an `assets` array, use it
      - If response has a `data` property with assets, try that
      - If nothing works, log the raw response keys to stderr and return a helpful error
   c. Map each asset to `{ id, name, type, active }` using defensive property access with `?.` and fallbacks
   d. If `type_filter` is provided, filter the list (case-insensitive substring match)
   e. Format output:
      ```
      Scene Assets ({count}):
      - {name} (type: {type}) [ID: {id}]{active ? "" : " [inactive]"}
      ```
   f. If no assets found (or after filtering): return "No assets found in the current scene." or "No assets matching type '{filter}' found."
   g. Wrap everything in try-catch, return `warudoError()` on failure

IMPORTANT:
- This tool uses the WebSocket client (wsClient), NOT the REST client
- The WebSocket action name "getScene" is based on research and may need adjustment when tested against a running Warudo instance. The error message should mention this possibility.
- Use defensive parsing -- the response structure is not fully documented
- Log unexpected response structures to stderr for debugging: `console.error("list_assets raw response keys:", Object.keys(response))`
  </action>
