**Step 2: Add get_asset_details tool to src/index.ts**

Register tool after the list_assets tool:
```
server.tool(
  "get_asset_details",
  "Read data input port values for a specific asset by entity ID. Use list_assets first to find entity IDs.",
  {
    entityId: z.string().describe("The entity ID (UUID) of the asset to inspect. Get IDs from list_assets."),
    port_key: z.string().optional().describe("Specific data input port key to read. Omit to get all ports."),
  },
  handler
)
```

Handler implementation:

**Strategy A (preferred): Use getScene and filter**
1. Call `wsClient.send({ action: "getScene" })` to get full scene data
2. Find the asset matching `entityId` in the response's asset list
3. If not found: return `warudoError("Asset not found with ID: {entityId}\n\nUse list_assets to see available assets and their IDs.")`
4. Extract data input ports from the asset data (likely under a `dataInputs` or similar property)

**Strategy B (alternative): Use getEntityDataInputPortValue**
If a specific WebSocket action exists for reading individual port values:
1. Call `wsClient.send({ action: "getEntityDataInputPortValue", entityId, dataInputPortKey: port_key })`
2. This may be more efficient for single-port reads

**Implementation notes:**
- Try Strategy A first since it's more reliable (reuses the same pattern as list_assets)
- If Warudo's response includes port values in the scene dump, Strategy A handles both the "all ports" and "specific port" cases
- If ports are NOT included in the scene dump, fall back to Strategy B for individual port reads
- The implementation should try the approach that works and log to stderr if the response structure is unexpected

**Formatting:**
- When returning all ports:
  ```
  Asset: {name} ({type}) [ID: {id}]
  Active: {yes/no}

  Data Input Ports ({count}):
    {portKey}: {value}
    {portKey}: {value}
    ...
  ```
- When returning a specific port:
  ```
  Asset: {name} ({type}) [ID: {id}]
  Port: {port_key}
  Value: {formatted value}
  ```
- For complex values (objects, arrays), use JSON.stringify with 2-space indent
- Truncate very long values (>500 chars) with "... (truncated)" to avoid overwhelming AI context

**Error messages must be actionable:**
- Not found: "Asset not found with ID: {id}. Use list_assets to see available assets."
- Port not found: "Port '{key}' not found on asset '{name}' ({type}). Available ports: {list first 20 port names}"
- Connection failure: "Failed to read asset details: {error message}"

IMPORTANT:
- This tool depends on 02-01's list_assets working correctly -- the asset parsing logic should be consistent
- Reuse any helper functions or patterns established in 02-01
- Log raw response structure to stderr on first call to aid debugging
